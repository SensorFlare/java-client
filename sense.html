Accessing Resource Data

The api calls described in this section can be used to retrieve information about user resources (Sensors and Actuators) as well as all the metadata related to them.


List All User Resources

GET /api/resource

Returns a JSONArray with all the IoT Resources accessible by the user (owned or shared with) and their user-friendly names when available.

Example:

[
 {
  resourceUri1: "resourceName1"
 },
  resourceUri2: "resourceName2"
 },
  ...
 ]

Get Resource Description

GET /api/resource/{RESOURCE_URI}/description

Returns a JSONArray with the description of the specified IoT Resource.

Example:

{
  uom: "Uom",
  isa: "Sensor",
  observes: "Atmospheric Pressure"
}

Get Latest Summary of Resource

GET /api/resource/{RESOURCE_URI}/report/latest

Returns a JSONObject with the latest information concerning the IoT Resource.

Example:

{
  keyName: "Keyname",
  latestTime: "2014-05-04T16:57:52.999Z",
  timeZone: "UTC",
  latest: 0,
  minutes5: [
   0,
   0,
   ...
  ],
  minutes60: [
   0,
   0,
    ...
  ],
  day: [
   0,
   0,
    ...
  ],
  daycost: 0,
  month: [
   0,
   0,
    ...
  ],
  monthcost: 0,
  max: {
    day: 0,
    month: 0
    }
  min: {
    day: 0,
    month: 0
    }
  average: {
    day: 0,
    month: 0
  },
  latestString: "Latest",
  minutes5Cost: 0,
  minutes60Cost: 0
}

Clear all properties of a Resource

GET /api/resource/clear/**

Returns {"status":"OK","code":200} if all properties are suceessfully removed from the Resource, otherwise returns "error".
