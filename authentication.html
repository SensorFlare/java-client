Java Client

Accessing Private User Data

Requests that require authentication will return {"status":"not found","code":404} in some cases instead of {"status":"forbidden","code":403}. This is to prevent the accidental leakage of private resources to unauthorized users.

Basic Authentication

Using curl: 
$ curl -u foo:bar http://www.sensorflare.com/api/dashboard
Using Java: 
HttpURLConnection con = null;
try {
    //Create connection
    URL url = new URL("http://www.sensorflare.com/api/dashboard");
    con = (HttpURLConnection) url.openConnection();
    con.setRequestMethod("GET");
    BASE64Encoder enc = new sun.misc.BASE64Encoder();
    String userpassword = username + ":" + password;
    String encodedAuthorization = enc.encode(userpassword.getBytes());
    con.setRequestProperty("Authorization", "Basic " +
    con.setRequestProperty("User-Agent", "sensorflare-java-client");
    encodedAuthorization);
    System.out.println("ResponseCode: " + con.getResponseMessage());
    BufferedReader in =
        new BufferedReader(new InputStreamReader(con.getInputStream()));
    String line;
    while ((line = in.readLine()) != null) {
        System.out.println(line);
    }
} catch (Exception e) {
    e.printStackTrace();
} finally {
    if (con != null) {
        con.disconnect();
    }
}

Authentication Errors

When user authentication fails appropriate error messages are provided to the client. 
{
    "timestamp":1398760457973,
    "error":"Unauthorized",
    "status":401,
    "message":"Bad Credentials"
}
