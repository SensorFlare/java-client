Sensorflare API reference

Download the Sensorflare Library as zip file Download the Sensorflare Library as tar.gz file

Sensorflare offers a REST API. Rest means that the urls used are properly describing the actions and interactions performed.

An overall categorization of the different interactions can be achieved using the following sections:

Data

used to retrieve information about sensors and the state of actuators.

Control

used to interact with IoT and Web resources.

Push

used to register clients (e.g., mobile phones) for notifications in the event of any change to the values of the resources.

AAA

used to authenticate users and manage permissions on actions and resources.
